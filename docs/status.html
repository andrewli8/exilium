Two invariants hold everywhere: the MCP layer serves cached local data only and can never trigger an upstream request (agents can't spend the API quota), and nothing in the system executes a trade — plans end at the human.

Surfaces · four ways in, one engine

exilium — Terminal UI (default)

Full-screen Ink app: MOVERS with 7-day sparkline detail, OPPORTUNITIES feed, ARBITRAGE table, ←→ category cycling. Refetches upstream every 5 min on its own with a freshness dot — built to live on a second monitor while you play.

exilium <cmd> — CLI

ingest · watch · snapshot · opps · arb · price · dashboard · mcp — aligned tables straight to the terminal.

MCP server — for AI agents

get_leagues · get_market_snapshot · get_pair_history · price_item · find_opportunities · find_arbitrage · get_categories · list_items · draft_trade_plan · create_watch · list_watches · delete_watch · poll_watch_results. Registered and connected in Claude Code.

Watch mode — get pinged

Refreshes on an interval, notifies once per new signal above your edge floor: desktop notification, terminal rationale, optional Discord webhook.

Signal engine

SignalMethodHonest status
mean-reversion Z-score of the latest daily change vs the trailing 7-day window; volume-scaled confidence; gold-fee-aware trade plans. Live — headline detector. ~40 signals on current PoE1 league data.
cross-rate divergence Listed price vs price implied by the highest-volume quote pair and core rates (two-leg consistency check). Live as the arb view. Markets are efficient — gaps run <0.5% in practice; wide gaps surface at league starts and on thin volume.
bulk↔single spread Exchange bulk price vs individual listing price. Blocked — poe.ninja retired listing-based price APIs; needs a listing data source.

Compliance anchors · non-negotiable

  1. 01 — Human-in-the-loop execution: no trade automation, ever. Every MCP tool description carries the rule.
  2. 02 — Durable edges, not latency races: data is minutes old and execution is human, so signals target inefficiencies that persist for hours.
  3. 03 — MCP serves cached data only: agents can never trigger upstream calls or burn rate limits.
  4. 04 — No POESESSID server-side, ever: no session cookies, no game-file access, polite User-Agent identifying the tool.
